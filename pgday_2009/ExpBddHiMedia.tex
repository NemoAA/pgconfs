\subsection{Outils, Méthodes}

\frame{
  \frametitle{Matériel} 

  Nos instances fonctionnent sur du materiel classique et récent : 

  \begin{itemize}
    \item<2-> Les grosses instances ont une machine dédiée.
    \item<3-> Le paramètrage y est adapté
    \item<4-> Certaines « petites » instances partagent le materiel
  \end{itemize}

}

\frame{
  \frametitle{Système} 

  \begin{itemize}
    \item<1-> Toutes les instances fonctionnent sous \texttt{Debian GNU/Linux}, et parfois \texttt{Centos}.
    \item<2-> Les versions de \texttt{PostgreSQL} viennent de la distribution ou des « \textit{backports} ».
    \item<3-> Les outils connexes sont empaquetés si nécessaire.
  \end{itemize}

}

\frame{
  \frametitle{Monitoring} 

  Les outils de monitoring sont les mêmes pour l'ensemble de la plateforme :
  \texttt{Nagios} et \texttt{Munin}.
  \linebreak
  \linebreak
  \pause

  Des scripts spécifiques sont mis en place pour monitorer les bons indicateurs.

}

\frame{
  \frametitle{pgFouine} 

  \texttt{pgFouine} est un analyseur de logs générant des rapports HTML : 

\begin{itemize}
    \item<2-> Les logs générés par chaque instance sont analysés par pgfouine
    \item<3-> Les rapports sont consultés chaque jour par les DBAs et les chefs de projets.
    \item<4-> De nombreuses requêtes ont été améliorés ou réécrites
\end{itemize}

}


\subsection{Performances}

\section{Outils avancés, réplication, plan de reprise}

\subsection{Skytools}

\frame{
  \frametitle{Réplication, Queuing}

  Skype fournit les outils développés en interne, nous les utilisons :

  \begin{definition}
    \alert{\texttt{londiste}} Réplication maître-esclave asynchrone

    \alert{\texttt{PGQ}} Queuing pour nos traitements par lots

    \alert{\texttt{WalMgr}} WAL Shipping (\textit{plan de reprise d'activité})

    \alert{\texttt{pgbouncer}} pool de connexions (\textit{prepared statements!})

    \alert{\texttt{plproxy}} distribution de données
  \end{definition}
}

\frame{
  \frametitle{Pool de connections}

  pgBouncer est utilisé devant toutes les instances.
  \linebreak
  \linebreak

  Il permet de protéger les instances derrière les scripts \texttt{PHP}, et
  sur les plateformes de développement et de pré-production, de maintenir
  plusieurs copies de la production en parallèle.

} 

\subsection{Réplications}

\frame{
  \frametitle{Réplication : différents usages}

  Selon les projets l'outil de réplication sert différents besoins :

  \begin{itemize}
    \item<2-> Séparer des couches applicatives autonomes
    \item<3-> Fédérer les « \textit{logs} » applicatifs sur une base centrale
    \item<4-> Publier sur des \textit{frontaux} les informations du \textit{backoffice}
    \item<5-> Distribuer l'analyse des données produite et la préparation des vues matérialisées
    \item<6-> Maintenir un réplica de la production pour \textit{failover}
  \end{itemize}
}

\subsection{Distribution de données}

\frame{
  \frametitle{Distribution de données avec \texttt{plproxy}}

  Une de nos applications offre un cas très adapté à l'utilisation de
  \texttt{plproxy} :

  \begin{itemize}
    \item<2-> Affichage de données consolidées par \textit{compte}
    \item<3-> Près de \texttt{300000} \textit{comptes} actifs
    \item<4-> Volumes de données de plus en plus importants
    \item<5-> \texttt{plproxy} nous permet de distribuer les données sur
      autant de partitions que nécessaire
  \end{itemize}

}

\subsection{Traitements par lots}

  \linebreak

\subsection{La communauté et PgFoundry}

\frame{
  \frametitle{Utiliser les contributions proposées}

  Malgré l'étendu des fonctionnalités de PostgreSQL, PgFoundry fournit des
  outils supplémentaires
  \pause

  \begin{definition}
    \alert{\texttt{ip4r}} stocker et indexer des adresses (et plages) IP

    \alert{\texttt{temporal}} stocker et indexer des plages de temps

    \alert{\texttt{pgtrgm}} recherche par correspondances de tri-grammes
  \end{definition}
}

\frame{
  \frametitle{Contribuer en retour}

  \begin{itemize}
    \item<1->Lorsque les fonctionnalités internes et les contributions ne suffisent pas, il faut retrousser les manches : \texttt{prefix}, \texttt{pg\_staging}
    \item<2-> \texttt{prefix} : stocker et indexer des préfixes de numéros de téléphones.
    \item<3-> extension écrite en C, avec l'aide de développeurs de PostgreSQL, sous licence \texttt{BSD}
    \item<4-> \texttt{pg\_staging} ; rendez-vous demain 14h10 \textit{English, developer}
  \end{itemize}
}

\section{Conclusion}

\subsection{Une solution robuste}

\frame{

  Très peu de pannes liés aux bases de données.

\begin{itemize}
    \item<2-> Quelques plantages (\texttt{SEGFAULT}), toujours dûes aux extensions (\textit{skytools}, \textit{temporal}, ...)
    \item<3-> Le serveur parvient très souvent à redémarrer seul : \textit{downtime} très bas
    \item<4-> Très bonnes réactivité des développeurs ( \texttt{core} et/ou extensions ) pour comprendre et corriger
\end{itemize}

}

\frame{
  \frametitle{Le Bon Outil \textit{(tm)}}

  En conclusion, \alter{PostgreSQL} est un très bon outil, riche en
  fonctionnalité, extensible, avec une communauté d'utilisateurs et de
  développeurs riche et réactive, parfaitement utilisable en entreprise.
  \linebreak
  \linebreak
  \pause

  Pour en tirer le meilleur il est cependant fortement conseillé de devenir
  un membre à part entière de la \textit{communauté}. C'est de la que vient
  le meilleur support et les outils. 
  \linebreak
  \linebreak
  \pause

  Pour devenir membre de la communauté, il suffit de considérer que l'on en
  fait déjà partie :)

}
